# Inbound link-graph sidecar (v2.5 #16) — design

**Date:** 2026-06-08
**Roadmap item:** `#16` — Inbound link-graph sidecar (milestone **v2.5.0a2**)
**Scope:** new `openzim_mcp/linkgraph/` package (builder + reader + schema) + new `openzim_mcp/cli/build.py` sub-CLI dispatched from `main.py` + an additive `get_inbound_links_data` data-layer method beside [`get_related_articles_data`](../../openzim_mcp/zim/structure.py) + `"inbound"` promotion in [`zim_links`](../../openzim_mcp/tools/zim_links.py). **No wire-format change to existing tools — additive only.** A new opt-in offline build artifact; absence is reported gracefully.

## Problem

[`zim/structure.py`](../../openzim_mcp/zim/structure.py) `get_related_articles` is outbound-only — it derives "related" from a page's *own* outbound link frequency. There is no way to ask "what links **to** this article." Inbound discovery was removed in v0.9.0 because a runtime reverse-edge scan over a multi-million-article archive is prohibitively expensive (every query would re-walk the whole archive).

The fix is to move that cost to an explicit, one-time **operator build step** that materializes the reverse-edge graph into a SQLite sidecar next to the archive, so runtime inbound becomes an indexed lookup. The sidecar is optional; when it is absent (or stale), inbound reports that cleanly rather than failing or serving wrong data.

## Scope decisions (settled in brainstorming)

| Fork | Decision |
| --- | --- |
| **Spec scope** | Full vertical slice in one spec → one plan → ship as **v2.5.0a2**: build CLI + sidecar format + runtime inbound read + `zim_links` `"inbound"` enum promotion + graceful absence. The pieces only deliver value end-to-end. |
| **Inbound ordering** | Rank linking pages by their **own inbound-degree** (importance proxy), so authoritative/hub referrers surface first. Costs one `GROUP BY` aggregation at build time and a per-node `inbound_degree` column. |
| **Staleness** | **Strict.** A sidecar whose stored archive UUID + schema version doesn't match the live archive is treated exactly like a missing sidecar (same structured "rebuild" error). Never serve stale/incorrect edges. |
| **Storage/schema** | SQLite, **integer-keyed** edges (`nodes` map id↔path, `edges(target_id, source_id)`) with **build-time precomputed** `inbound_degree`. Smallest file, fast indexed reads, clean `LIMIT/OFFSET` pagination. |
| **Anchor text** | **Not stored** in a1 (keeps edges compact integer pairs). Inbound results carry `path` + `title` + `inbound_degree`, not the source's anchor text. Deferred enhancement. |

## Architecture

Six units, each independently testable, wired lowest-blast-radius first (schema → builder → reader → data method → async wrapper → tool).

### 1. Schema — `openzim_mcp/linkgraph/schema.py`

Pure DDL + constants, no I/O. `SCHEMA_VERSION` is an int bumped on any incompatible layout change (the reader rejects mismatches → forces a rebuild).

```sql
CREATE TABLE meta   (key TEXT PRIMARY KEY, value TEXT) STRICT;
  -- rows: schema_version, archive_uuid, built_at (UTC ISO8601),
  --       node_count, edge_count, builder_version
CREATE TABLE nodes  (id INTEGER PRIMARY KEY, path TEXT NOT NULL UNIQUE,
                     inbound_degree INTEGER NOT NULL DEFAULT 0) STRICT;
CREATE TABLE edges  (target_id INTEGER NOT NULL, source_id INTEGER NOT NULL) STRICT;
CREATE INDEX edges_by_target ON edges(target_id);
```

Inbound query (the only read path):

```sql
SELECT n.path, n.inbound_degree
  FROM edges e JOIN nodes n ON n.id = e.source_id
 WHERE e.target_id = :target_id
 ORDER BY n.inbound_degree DESC, n.path ASC
 LIMIT :limit OFFSET :offset;
```

Deterministic ordering (`inbound_degree DESC, path ASC`) makes offset pagination stable. `total` = `COUNT(*)` over the same `WHERE`.

### 2. Builder — `openzim_mcp/linkgraph/builder.py`

`build_link_graph(archive_path, out_path, *, force, progress) -> BuildStats`. Single pass over content (`C`-namespace) article entries, then a degree pass:

1. Open the archive; record `archive.uuid`, `entry_count` ([`zim/archive.py:630-632`](../../openzim_mcp/zim/archive.py)).
2. **Write to a temp file** `out_path + ".tmp"` (same directory, for atomic rename). Open SQLite with `PRAGMA journal_mode=OFF; PRAGMA synchronous=OFF` (safe — a crash just discards the temp file).
3. For each content entry (source): extract its **internal** outbound links by reusing the internal-bucket path of [`extract_article_links_data`](../../openzim_mcp/zim/structure.py) (or its underlying extractor), **resolve each href to its canonical target entry path** by following redirects ([`zim/redirects.py`](../../openzim_mcp/zim/redirects.py)) so reverse edges key on canonical entries; reject self-links; dedup `(source, target)` within a source.
4. Intern each path to a compact integer id via an in-memory `dict[str, int]`; `executemany`-insert `(target_id, source_id)` rows in batches inside one transaction; insert nodes lazily as ids are minted.
5. **Degree pass:** `UPDATE nodes SET inbound_degree = (SELECT COUNT(*) FROM edges WHERE target_id = nodes.id)` (or a `GROUP BY` build of a temp aggregate then update — whichever benchmarks faster); create `edges_by_target`.
6. Write `meta` rows; `COMMIT`; close; **atomic `os.replace(tmp, out_path)`**.

`BuildStats` (node/edge counts, elapsed, bytes) is returned for the CLI summary. The `path→id` dict is the only large in-memory structure (§ Risks). Builder never mutates the archive and is idempotent given `--force`.

### 3. Reader — `openzim_mcp/linkgraph/reader.py`

`LinkGraphReader.open_for(archive_path, *, live_archive_uuid) -> LinkGraphReader | None` and `query_inbound(target_path, limit, offset) -> InboundPage`.

- **Discovery:** sidecar path is derived from the validated archive path: `<archive>.zim.linkgraph.sqlite` (sibling file). Absent file → `open_for` returns `None` (caller emits the structured "build it" error).
- **Strict fingerprint:** on open, read `meta.schema_version` and `meta.archive_uuid`; if `schema_version != SCHEMA_VERSION` or `archive_uuid != live_archive_uuid` → return `None` (treated as absent; caller's error message notes "rebuild"). Open `mode=ro` (read-only URI) so the reader never writes/creates.
- `query_inbound` resolves `target_path → id` (missing → empty page, **not** an error — a valid entry with zero inbound), runs the ranked paginated query, returns rows `[{path, inbound_degree}]` + `total`. Title resolution (`path → archive title`) reuses the existing outbound title-resolution helper so inbound matches outbound's `path`/`title` shape.

### 4. Data layer — `get_inbound_links_data` (in `zim/structure.py`)

`get_inbound_links_data(zim_file_path, entry_path, limit=10, offset=0, *, cursor_archive_identity=None) -> RelatedArticlesResponse`-shaped dict. Validates path + `reject_path_traversal`; opens the archive to get the live UUID; calls the reader. Returns the Phase-B five-key contract (`results` / `next_cursor` / `total` / `done` / `page_info`) + `entry_path`, where each `results` item is `{path, title, inbound_degree}`. Inbound **paginates** (an article can have thousands of inbound links): it emits `next_cursor` via `Cursor.encode(tool="get_inbound_links", state={o, l, ep, ai})` ([`pagination.py`](../../openzim_mcp/pagination.py)). Sidecar absent/stale → raises a typed "sidecar unavailable" condition the tool turns into a structured error (below). An async wrapper is added beside [`get_related_articles_data`](../../openzim_mcp/async_operations.py).

### 5. CLI — `openzim_mcp/cli/build.py`, dispatched from `main.py`

`main.py` gains a second early-`sys.argv[1]` check mirroring the `download-models` one ([`main.py:113-116`](../../openzim_mcp/main.py)):

```python
if len(sys.argv) >= 2 and sys.argv[1] == "build":
    from openzim_mcp.cli.build import build_main
    sys.exit(build_main(argv=sys.argv[2:]))
```

`build_main` parses an **artifact subcommand** so `build` is a namespace (leaving room for `build embeddings` under sub-D-4): `openzim-mcp build link-graph <archive.zim> [--force] [--output PATH] [--quiet]`. It routes `link-graph` to `linkgraph.builder.build_link_graph`, prints progress to stderr (throttled: entries walked / edges written), and on success prints the sidecar path, node/edge counts, file size, and elapsed time. Honors the same `OPENZIM_MCP_*` env conventions as [`ml/cli/download.py`](../../openzim_mcp/ml/cli/download.py). Exit 0 on success, nonzero (with a clear message) on a missing/corrupt/locked archive or an existing sidecar without `--force`.

### 6. Tool wiring — `zim_links`

Promote `"inbound"` into `_VALID_DIRECTIONS` and the `Literal` ([`zim_links.py:20,33`](../../openzim_mcp/tools/zim_links.py)); add an `inbound` branch that decodes its cursor with `expected_tool="get_inbound_links"` (reusing this session's `decode_cursor_state` / `cursor_context_mismatch` in [`tools/_common.py`](../../openzim_mcp/tools/_common.py)) and calls the async inbound method. The description's "`inbound` lands in v2.5" note ([`zim_links_description.md`](../../openzim_mcp/tools/zim_links_description.md)) becomes real documentation: inbound behavior, that `cursor`/`offset` apply to it, and the **"requires a built link-graph sidecar"** precondition. `outbound`/`related` are unchanged.

## Runtime data flow

`zim_links(direction="inbound", entry_path=X, [cursor|offset], limit)` → decode/validate cursor → `AsyncZimOperations.get_inbound_links_data` → `get_inbound_links_data` → `LinkGraphReader.open_for(archive, live_uuid)`:

- **None** (absent or stale) → `tool_error(operation="inbound_sidecar_unavailable", …)` naming the exact build command.
- **Reader present** → ranked paginated query → title-resolve sources → five-key response.

## Error handling & edge cases

| Case | Behavior |
| --- | --- |
| Sidecar file absent | Structured `tool_error` (not a crash): "Inbound links require a link-graph sidecar. Run `openzim-mcp build link-graph <archive>`." |
| Sidecar stale (UUID or schema mismatch) | Same error, message notes the sidecar is out of date and must be rebuilt. Never serves stale edges. |
| Target entry has zero inbound links | Empty `results`, `total: 0`, `done: true` — **not** an error. |
| Target path not in `nodes` (e.g. non-content entry) | Same empty page. |
| Foreign / malformed / wrong-entry cursor | Existing cursor guards (`cursor_mismatch` / `cursor_context_mismatch` / `cursor_decode`) from this session. |
| Build: sidecar exists, no `--force` | Refuse with a clear message; `--force` overwrites. |
| Build interrupted | Temp file discarded; `out_path` never half-written (atomic rename). |
| Build: corrupt/locked archive | Nonzero exit + message; no partial sidecar. |

## Risks (flagged in brainstorming)

- **Schema-byte budget.** Promoting `"inbound"` into the enum + expanding the `zim_links` description grows the advanced tool-schema, which sits ~64 bytes under the hard **24,500** cap after this session's trim ([`test_phase_f_schema_budget.py`](../../tests/test_phase_f_schema_budget.py), `test_phase_f_prototype_parity.py`). The implementation must keep the `zim_links` description lean and, if needed, redistribute the per-tool `ALLOCATION` (the test permits this — total is the only hard cap) and refresh the prototype-parity snapshot. **Treated as an acceptance gate, not an afterthought.**
- **Build-time memory.** The `path→id` dict over a multi-million-entry archive is a few hundred MB of RAM on the operator build box. Acceptable for an explicit build step; the spec keeps it the only large in-memory structure (edges stream to SQLite in batches). If it ever proves too large, a follow-up can spill the interning map to a temp SQLite table — out of scope for a1.

## Testing strategy

- **Unit (no ZIM):** schema DDL applies; builder invert + self-link rejection + `(source,target)` dedup + redirect resolution (fed canned link lists); degree computation; reader fingerprint match vs mismatch vs missing-file; pagination offset/cursor round-trip; zero-inbound empty page.
- **Integration (small fixture archive):** `build_link_graph` produces a sidecar; inbound query returns the expected sources in `inbound_degree DESC` order; staleness (mutate `meta.archive_uuid`) → treated as absent; graceful-absence (no sidecar) → structured error.
- **Tool:** `zim_links` accepts `inbound`; cursor wiring + mismatch guards; description fidelity test (the `tools/_common` description⇄schema guard pattern).
- **Budget gates:** `test_phase_f_schema_budget` + `test_phase_f_prototype_parity` stay green (see Risks).
- **CLI:** `build link-graph` happy path, `--force` overwrite, refuse-without-force, missing-archive nonzero exit.

## Backward compatibility & non-goals

Purely additive: a new CLI subcommand, a new optional sidecar file, a new `direction` enum value, and a new data method. No existing tool signature, response key, or behavior changes; `outbound`/`related` are untouched. **Non-goals:** semantic/embedding retrieval (sub-D-4), incremental sidecar updates, cross-archive inbound, anchor-text storage, and any change to the locked v2 wire surface beyond the additive `inbound` enum value.
