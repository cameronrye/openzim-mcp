# Archive-type presets (v2.5 #17) ‚Äî design

**Date:** 2026-06-04
**Roadmap item:** `#17` ‚Äî Archive-type presets (milestone **v2.5.0a1**, first v2.5 pre-release)
**Scope:** new `openzim_mcp/archive_types.py` + `openzim_mcp/preset_data.py` + `openzim_mcp/data/presets.toml` + additive seams in `zim/archive.py`, `content_processor.py`, `zim/content.py`, `config.py`. **No tool-surface or response-contract change ‚Äî additive only.**

## Problem

Wikipedia, Wiktionary, Stack Exchange, and TED-style ZIMs have different shapes (encyclopedic prose vs. dictionary glosses vs. Q&A vs. transcripts), but the server treats every archive identically. The server's current defaults are implicitly Wikipedia-shaped (lead-paragraph snippet, first-section summary). For a Q&A archive a "summary" should be the accepted answer, not the first prose block; models waste turns rediscovering archive shape on every non-Wikipedia corpus.

There is no archive-type notion in the codebase today, no `presets/` data, and no per-archive config. This item adds all three, scoped tightly for the first alpha.

## Scope decisions (settled in brainstorming)

| Fork | Decision |
| --- | --- |
| **a1 behavior scope** | Snippet shape + summary style only. Detect-and-report underneath. The default-namespace seam and intent-parser priors are **deferred** (see Non-goals). |
| **Types** | The classifier detects all four types + `generic`. Behavior presets attach to **`wikipedia`** (explicit baseline) and **`stackexchange`** (the real change) only. `wiktionary` / `ted` are detected and reported but have no preset entry ‚Üí generic behavior; a2 adds their entries as pure data. |
| **Detection** | Deterministic rule-cascade over high-signal `M`-namespace fields, emitting a coarse confidence tier (`high` / `medium` / `none`). The behavior preset is applied **only at `high`** confidence (or when a per-archive pin forces it). Plain string matching, no regex, never raises. |
| **Storage** | Bundled `openzim_mcp/data/presets.toml` keyed by type + `OPENZIM_MCP_PRESETS_OVERRIDE_PATH` deep-merge + optional per-archive pin keyed by `M/Name`. |
| **Surfacing** | Additive `_meta.detected_type` (+ confidence) and `_meta.preset_applied`. No contract change. |
| **Synthesize** | `synthesize` stays on the global-default snippet shape. Presets shape only direct-search snippets and `zim_get(view='summary')`. |

## Architecture

Five units, each independently testable, wired lowest-blast-radius first.

### 1. Detection ‚Äî `openzim_mcp/archive_types.py` (pure function, no I/O)

```python
ArchiveType = Literal["wikipedia", "wiktionary", "stackexchange", "ted", "generic"]
Confidence = Literal["high", "medium", "none"]

def detect_archive_type(metadata: Mapping[str, Any]) -> tuple[ArchiveType, Confidence]: ...
```

- Input is the already-built metadata dict (the `M`-namespace `metadata_entries` plus structural fields) ‚Äî **no archive handle, no libzim API**. Trivially unit-testable against plain fixture dicts; no ZIM build required.
- Rule-cascade over high-signal fields, most-specific first:
  1. `Scraper` ‚Äî strongest unique signal: `startswith("mwoffliner")` ‚áí wiki family (then disambiguate wikipedia vs wiktionary via `Name`/`Title`), `startswith("sotoki")` ‚áí `stackexchange`, `startswith("ted2zim")` ‚áí `ted`.
  2. `Name` prefix ‚Äî `wikipedia_`, `wiktionary_`, `ted_`, or a Stack Exchange site host (`*.com_`, `*.stackexchange.com_`).
  3. `Creator` / `Tags` ‚Äî corroborating only (e.g. `Tags` containing `wikipedia`, `Creator` containing `Stack Exchange`).
- **Confidence tiers:** `high` = a strong unique signal matched (e.g. `Scraper` startswith `sotoki`, or `Scraper`+`Name` agree). `medium` = only a weak corroborating signal matched (e.g. `Tags`/`Creator` alone, or `Name` prefix without a `Scraper`). `none` = no match ‚áí type `generic`.
- All matching is plain `str.startswith` / `in` (case-folded), **bounded, no regex** ‚Äî satisfies the ReDoS / Sonar S5852 gate.
- **Graceful fallback:** missing keys, empty metadata, or old-scheme archives ‚áí `("generic", "none")`. The function never raises and never does I/O.

**Where it runs:** once inside `ZimOperations._extract_zim_metadata` ([`zim/archive.py:613-675`](../../openzim_mcp/zim/archive.py)), immediately after `metadata_entries` is populated. It writes two reserved keys onto the returned dict:

```python
metadata["_detected_type"], metadata["_detection_confidence"] = detect_archive_type(metadata)
```

The classifier reads its signals from `metadata.get("metadata_entries", {})` (the subdict is absent when an archive has no readable `M`-namespace entries ‚Äî see [`zim/archive.py:664-665`](../../openzim_mcp/zim/archive.py)), so it must tolerate a missing `metadata_entries`.

Detection is **deterministic on metadata-only inputs**, so the existing response-level cache (`metadata_data:v2c:{validated_path}`) stays correct with no key change ‚Äî the detected type is memoized for free alongside the metadata it derives from. The reserved keys are internal (leading underscore). Because `_extract_zim_metadata` returns the dict that the metadata tool partly serializes (`uuid`, `is_multipart`, etc. are top-level keys on it ‚Äî [`zim/archive.py:627-630`](../../openzim_mcp/zim/archive.py)), **the metadata response builder must exclude `_`-prefixed reserved keys from the public body** and surface the detected type only via `_meta` (¬ß5). They are never added to the public `metadata_entries` map.

### 2. Preset data layer ‚Äî `openzim_mcp/preset_data.py` + `openzim_mcp/data/presets.toml`

**No new dependency.** `requires-python = ">=3.12"` ([`pyproject.toml:6`](../../pyproject.toml)) ‚áí stdlib `tomllib`. Files are opened in **binary mode** (`tomllib.load` requires it and decodes utf-8 internally ‚Äî sidesteps the Windows cp1252 trap from project memory).

**Schema** ‚Äî Pydantic `ArchivePreset`, consistent with the `BaseModel` config hierarchy ([`config.py:85-94`](../../openzim_mcp/config.py)), all fields Optional, `model_config = ConfigDict(extra="forbid")` to catch typos at load time:

```python
class ArchivePreset(BaseModel):
    model_config = ConfigDict(extra="forbid")
    snippet_length: Optional[int] = Field(default=None, ge=100)
    max_paragraphs: Optional[int] = Field(default=None, ge=1)
    summary_style: Optional[Literal["first_section", "q_and_a"]] = None
```

`"gloss"` (Wiktionary) is reserved for a2 ‚Äî added to the `Literal` when its behavior lands. A missing field means "inherit": preset default ‚Üí global config default. Validation happens **eagerly at load**, not lazily at use (avoids a server-startup-time surprise and the CodeQL `py/uninitialized-local-variable` trap).

**Loader** ‚Äî mirrors the verified `query_rewrite_data.py` pattern ([`query_rewrite_data.py:21-67`](../../openzim_mcp/query_rewrite_data.py)): module-level functions, `@functools.lru_cache`, `path=None` ‚áí bundled default via `importlib.resources.files("openzim_mcp.data") / "presets.toml"`.

```python
@functools.lru_cache(maxsize=8)
def load_presets(override_path: Optional[Path]) -> PresetSet: ...
```

`PresetSet` holds the per-type presets plus the per-archive pins (see below). `pyproject.toml` package-data is extended: `openzim_mcp = ["data/*.txt", "data/*.toml", "tools/*.md"]`.

**Bundled `presets.toml` (a1)** ‚Äî only the two types that get behavior:

```toml
[preset.wikipedia]            # explicit baseline == current defaults; makes the baseline testable
summary_style = "first_section"

[preset.stackexchange]        # the real behavior change; exercises both wired seams
summary_style = "q_and_a"
max_paragraphs = <pinned>     # concrete value pinned during implementation via live superuser probes
```

`wiktionary` / `ted` have **no `[preset.*]` table** ‚áí they resolve to no preset ‚áí generic behavior, while still being detected and reported.

**Override + per-archive pins** ‚Äî `OPENZIM_MCP_PRESETS_OVERRIDE_PATH` (free via the existing `env_prefix="OPENZIM_MCP_"`) points to a TOML that is **deep-merged per type** over the bundled defaults, plus optional `[archive."<name>"]` pins keyed by `M/Name`:

```toml
[preset.stackexchange]                       # deep-merge: tweak one field, rest inherited
max_paragraphs = 2

[archive."superuser.com_en_all_2026-02"]     # per-archive pin, keyed by M/Name (stable, path-independent)
type = "stackexchange"                        # force a type (overrides the confidence gate)
summary_style = "first_section"               # per-archive field override
```

A pin's `type` forces preset application for that specific archive even at `medium`/`none` confidence ‚Äî the escape hatch for mis-detection. Pin field overrides win over the type preset, which wins over the bundled default, which falls back to the global config default. One inheritance rule, stated once.

### 3. Resolution glue ‚Äî `resolve_preset` (downstream of the cache)

```python
def resolve_preset(config: OpenZimMcpConfig, metadata: Mapping[str, Any]) -> Optional[ArchivePreset]: ...
```

- Reads the cached `_detected_type` / `_detection_confidence` from `metadata`.
- Looks up the per-archive pin (by `metadata.get("metadata_entries", {}).get("Name")`) first; if a pin forces a type, apply it. Otherwise apply the type preset **only when confidence is `high`**. `medium` / `none` (and no pin) ‚áí return `None` (generic behavior).
- **All override/pin resolution happens here, downstream of the metadata cache.** Editing the override file therefore never requires cache invalidation; only metadata-derived detection lives inside the cached `_extract_zim_metadata`. (This is the correctness boundary that keeps the path-keyed cache valid.)
- Resolved **once per operation** on a given archive; the resolved preset's fields are then threaded into per-entry snippet/summary calls (no per-entry re-resolution).

### 4. Wiring ‚Äî both no-ops under the generic/no-preset case

**(a) Snippet shape (LOW risk).** `ContentProcessor.create_snippet` ([`content_processor.py:970-1086`](../../openzim_mcp/content_processor.py)) and its caller `_get_entry_snippet` ([`zim/content.py:138-200`](../../openzim_mcp/zim/content.py)) gain optional per-call `snippet_length` / `max_paragraphs` arguments that fall back to the instance defaults (`self.snippet_length`, today `DEFAULT_SNIPPET_LENGTH = 3000` at [`defaults.py:42-60`](../../openzim_mcp/defaults.py)) when `None`. The direct-search path resolves the preset once for the searched archive and passes these through. When no preset applies, the single-value truncation path ([`content_processor.py:1055-1065`](../../openzim_mcp/content_processor.py)) is **bit-identical** to today.

**(b) Summary style (LOW‚ÄìMEDIUM risk).** Thread an optional `preset: Optional[ArchivePreset] = None` from `zim_get(view='summary')` ([`tools/zim_get.py:127-130`](../../openzim_mcp/tools/zim_get.py)) ‚Üí `AsyncZimOperations.get_entry_summary_data` ([`async_operations.py:509`](../../openzim_mcp/async_operations.py)) ‚Üí `get_entry_summary_data` ([`zim/content.py:1356`](../../openzim_mcp/zim/content.py)) ‚Üí `_extract_entry_summary_data` ([`zim/content.py:1444`](../../openzim_mcp/zim/content.py)). A `summary_style == "q_and_a"` branch is added **after** the MIME dispatch (~`:1467`) and selects the accepted-answer section **within the existing bundle** ([`bundle.py:356` `get_or_build_bundle`](../../openzim_mcp/bundle.py)) ‚Äî it must **never rebuild the bundle** (the bundle is shared by toc/structure/get_section; rebuilding pollutes that cache). `preset=None` or `summary_style="first_section"` ‚áí the existing first-section slice (~`:1510`) runs unchanged. The response shape (`EntrySummaryResponse`: title/path/summary/word_count/is_truncated) is **unchanged** ‚Äî only the summary *string* varies.

The exact sotoki section selector (which heading/landmark marks the accepted answer in the live superuser archive) is **pinned during implementation against the deployed owl-atlas server** ‚Äî the same deploy-and-reprobe discipline the tail-hijack spec used for its Step 1, because the live archive is the only source of truth for sotoki's HTML structure.

### 5. Surfacing ‚Äî additive `_meta`

Using the existing `MetaConfig` envelope ([`config.py:96`](../../openzim_mcp/config.py)):

- `_meta.detected_type` (+ `_meta.detection_confidence`) on the metadata response ‚Äî reported even at `medium` confidence, so a client can see the guess without a preset being applied.
- `_meta.preset_applied` (the type whose preset shaped the output) on shaped search/summary responses ‚Äî so a styled `q_and_a` summary is never mistaken for a truncation. Absent when no preset was applied.

Purely additive; no existing field changes.

## Data flow (end to end)

```
open archive ‚îÄ‚ñ∂ _extract_zim_metadata
                  ‚îú‚îÄ build metadata_entries (M-namespace)               [unchanged]
                  ‚îî‚îÄ detect_archive_type(metadata) ‚îÄ‚ñ∂ _detected_type/_confidence   [cached]
                                                       ‚îÇ
direct search / zim_get(summary) on archive X          ‚ñº
   resolve_preset(config, cached_metadata) ‚îÄ‚ñ∂ ArchivePreset | None
       ‚îÇ (high-confidence or pin-forced)        ‚îÇ
       ‚îú‚îÄ snippet_length/max_paragraphs ‚îÄ‚ñ∂ create_snippet(...)          [no-op if None]
       ‚îú‚îÄ summary_style ‚îÄ‚ñ∂ _extract_entry_summary_data(...)             [no-op if None/first_section]
       ‚îî‚îÄ type ‚îÄ‚ñ∂ _meta.preset_applied                                  [additive]
```

`synthesize` does **not** call `resolve_preset`; its internal passage extraction (`_extract_passages` ‚Üí `create_snippet`) keeps the global-default snippet shape, fully isolating RRF fusion, section-attribution, and the just-hardened tail-hijack / cross-archive-leak gates from preset variance.

## Testing

1. **Unit ‚Äî detection.** `detect_archive_type` over plain fixture dicts: each known type at `high` (strong signal), at `medium` (corroborating-only), `generic` at `none`, and graceful-fallback cases (empty dict, missing keys, old-scheme). Each fixture asserts an exact `(type, tier)`.
2. **Unit ‚Äî preset data.** Loader: bundled load, deep-merge override, per-archive pin (force-type + field override), malformed TOML / unknown key (`extra="forbid"` rejects) ‚Üí graceful. `resolve_preset`: confidence gating (`high` applies, `medium`/`none` ‚Üí `None`), pin forces past the gate, inheritance/fallback order.
3. **Integration.** Build typed test ZIMs (synthetic `M/Scraper`, `M/Name`, etc.) via the `tests/conftest_v2_fixtures.py` `_HtmlItem` / `make_zim_ops` helpers ([`:37-60`, `:147-163`](../../tests/conftest_v2_fixtures.py)). Exercise the snippet + summary seams end-to-end; assert the **generic path is bit-identical** to today (no preset ‚áí unchanged output).
4. **Invariant regression (must stay green).** `tests/test_post_v2_1_4_beta_fixes.py`, `tests/test_synthesize.py`: `Stalin USSR Russia`‚úó, `Einstein's theory ‚Üí Theory_of_relativity`‚úì, `Plato's cave ‚Üí Allegory_of_the_cave`‚úì, `ssh connection refused ‚Üí superuser`‚úì, `planet earth ‚Üí Earth`‚úì, `Berlin Germany ‚Üí Berlin`‚úì, `Tesla electricity ‚Üí Tesla's_Wireless_Electricity`‚úó.
5. **Live reprobe (owl-atlas ‚Äî the only end-to-end validator).** Stack Exchange `q_and_a` summary + SE search snippet on superuser; Wikipedia baseline unchanged; pin the sotoki accepted-answer selector here. Run before merge.

CI gates (project memory): full `make lint` (flake8+isort+black over `openzim_mcp` **and** `tests`, `max-line-length=128`), `mypy openzim_mcp` strict, no-ReDoS, CodeQL `py/uninitialized-local-variable`, Windows cp1252 (binary-mode TOML, no cwd-relative preset paths). New test files must be black/isort-clean from the first commit.

## Non-goals / deferred

- **Default-namespace seam** (HIGH risk: can silently delete hits; every known type is `C`-anyway). The injection point is `_scan_filtered_search` ([`zim/search.py:1652-1676`](../../openzim_mcp/zim/search.py)); deferred until a real non-`C` archive type exists. Shipped behavior: `default_namespace` is not in the a1 schema.
- **Intent-parser priors** (HIGHEST blast radius ‚Äî the just-hardened surface). Deferred; if ever added, priors may only be *additive and tightening*, applied symmetrically to both the `tell_me_about` and `synthesize` paths, never by mutating `INTENT_PATTERNS`.
- **`gloss` (Wiktionary) and TED transcript behavior** ‚Äî detected and reported in a1, behavior deferred to a2 (pure data + one `Literal` value).
- **Multi-archive preset arbitration under synthesize** ‚Äî synthesize stays global-default; revisit once single-archive presets are live-validated.
- No reranker, no semantic retrieval, no new dependency, no tool-schema or response-contract change.

## Build sequence (lowest blast radius first)

1. `archive_types.py` classifier + unit tests (pure; zero blast radius).
2. `preset_data.py` loader + `data/presets.toml` + `ArchivePreset` + `presets_override_path` config field + package-data + unit tests (data only).
3. `resolve_preset` glue + detection call inside `_extract_zim_metadata` + `_meta` surfacing.
4. Snippet-shape wiring (per-call overrides) + tests.
5. Summary-style `q_and_a` wiring + live sotoki-selector pin + tests.
6. Live reprobe, docs version-sync, ship **v2.5.0a1** pre-release.
